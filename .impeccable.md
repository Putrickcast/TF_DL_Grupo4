## Design Context

### Users

La demo la revisa un profesor de Deep Learning y la presenta un equipo de Data Science. El usuario operativo imaginado es un equipo comercial de administracion de departamentos tipo Airbnb en Barranco, Lima, que necesita decidir rapido si un listado conviene y por que.

### Brand Personality

Clara, analitica y confiable. Debe sentirse como una herramienta academica-profesional: evidencia primero, decisiones explicables, nada decorativo que parezca marketing.

### Aesthetic Direction

Dashboard claro en modo light, con densidad moderada y ritmo de herramienta de trabajo. El chat debe sentirse como una ventana conversacional central, no como una tarjeta secundaria. Las fotos reales deben aparecer despues del chat para sostener el flujo multimodal: preguntar, mirar evidencia visual, revisar criterios.

### Design Principles

- Priorizar evidencia y trazabilidad sobre adornos visuales.
- Mantener cada interaccion obvia: escribir, enviar, ver respuesta y revisar fragmentos.
- Evitar que el usuario confunda assets sinteticos con fotos reales.
- Usar secciones apiladas con jerarquia clara: chat, imagenes, criterios.
- Conservar una lectura rapida para presentacion academica y defensa tecnica.
